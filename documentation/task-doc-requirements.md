# Task-Dokumentation
- Inhalt: Durchführung eines ausgewählten Tasks oder User Story. Auswahl erfolgt in Rücksprache mit Lehrkraft! (Mögliche Tasks sind z. B. Datenbankdesign und Umsetzung, Anmeldevorgang, Auswahl eines Frameworks, Dashboard Design, Auswahl Monitoringframeworks)
- Abgabe der Dokumentation: spätestens Zeitpunkt Projektpräsentation
- Umfang: ca. 5 Seiten (PDF-Format)


## 1. Inhalte der Taskbearbeitung (Gewichtung 60%)

| Nr | Taskbeschreibung | Punkte |
| :--- | :--- | :--- |
| **1.1** | Ausführliche Beschreibung der Problemstellung | /10 |
| **1.2** | Beschreibung möglicher unterschiedlicher Lösungsmöglichkeiten (Konzepte) | /5 |
| **1.2** | Bewertung der beschriebenen Lösungsmöglichkeiten (Konzepte) | /10 |
| **1.3** | Fundierte Begründung des eingeschlagenen Lösungswegs | /10 |
| **1.4** | Planung der Sicherstellung der Qualität | /5 |
| **1.5** | Planung der Vorgehensweise (Ablaufdiagramme, Datenbankdesign, Mockup, etc.) | /10 |
| **1.6** | Umsetzung der Vorgehensweise (z. B. Quellcode, Konfigurationen, YAML-Dateien, ...) | /10 |
| **1.7** | Durchführung und Bewertung der Testergebnisse | /5 |
| **1.8** | Bewertung der Gesamtreflektion (Was war gut, was muss zukünftig besser werden) | /5 |
